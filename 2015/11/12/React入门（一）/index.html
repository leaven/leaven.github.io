初探React

Why use React

1、面向View层开发，一切皆组件
2、单向数据流
3、VirtualDom，高性能渲染
4、Learn once，write everywhere.(React Native for IOS、React Native for Android)
5、github star数量已破3W

Begin With React

1、require 'react'
2、require 'react-dom'
3、因为react语法需要jsx编译引擎，所以需要引入jsx编译脚本，传送门

现在我们在页面中渲染一个HelloWorld

var HelloWorld = React.createClass({
  render: function() {
    return (
        <div>
          HelloWorld
        </div>
    );
  }
});
ReactDOM.render(
  <HelloWorld />,
  document.getElementById("helloworld")
);

实际效果请戳这里
ReactDOM.render能将JSX模板编译成HTML插入到相应的element中。
注意：
1、自定义的类必须以大写字母开头，如果用小写字母会被认为是标准html元素，导致不能被解析**；
2、return返回必须需要dom标签包裹，否则会不能被解析；
3、JSX 的基本语法规则：遇到 HTML 标签（以 < 开头），就用 HTML 规则解析；遇到代码块（以 { 开头），就用 JavaScript 规则解析。

Props

Props是组件数据传递的载体，外部数据通过props将数据传递到组件内部，下面我们通过一个例子说明一下，源码地址

var HelloWorld = React.createClass({
  render: function() {
    return (
        <div>
          Hello,{this.props.name}
        </div>
    );
  }
});
ReactDOM.render(
  <HelloWorld  name="leaven"/>,
  document.getElementById('example')
);

实际效果戳这里
组件上可以增加任意属性（类似于我们在dom上添加data-*），组件内中可以通过this.props.*获取到组件上的自定义属性。

State

state是React中一个非常重要的概念，所有的UI改变都是通过改变state发生，state改变能够触发UI重绘，所以，所有的交互状态我们都应该尝试去改变state，这一点跟传统的dom交互大不一样，看下面的一个例子

var HelloWorld = React.createClass({
  getInitialState: function() {

    this.loadData();
    return {data: []};
  },
  loadData: function() {
    var me = this;
      $.getJSON(this.props.url, function(data) {
        me.setState({data: data});
      });
  },
  render: function() {
    var person = this.state.data.map(function(d, i) {
      return (
        <div className="name" key={i}>
          name:{d.name}, age:{d.age}
        </div> 
      )
    });
    return (
       <div>
          {person}
       </div>
    );
  }
});
ReactDOM.render(
  <HelloWorld url="./data/demo4.json"/>,
  document.getElementById('example')
);

源码地址
在请求到数据后，我们将state置为获得的data，导致render方法被执行。
注意
1、这里的map会返回一个数组，每一个元素都需要设置一个key，这里建议key设为数组的索引值，因为索引值能保证每个不一样；
2、关于state及props适用场景请参考这里。

事件监听

React中绑定事件监听类似于DOM 0级事件，看下面的例子

var HelloWolrd = React.createClass({
  getInitialState: function() {
      return {
          isName: false
      }
  },
  changeValue: function() {
      this.setState({"isName": (this.state.isName?false: true)});
  },
  render: function() {
        return (
          <div>
              Hello, {this.state.isName? 'leaven': 'world'}
              <button onClick={this.changeValue}>点击此处改变name</button>      
          </div>
        );
  }
});
ReactDOM.render(
  <HelloWolrd />,
  document.getElementById('example')
);

源码地址
注意
1、react中事件回调函数中的this会自动绑定到实例上，这里有个特殊情况，就是在es6中需要手动绑定。

获取DOM对象

获取组件内部的dom对象可以通过ref，this.refs是一个dom节点的引用数组，看下面的例子

var InputChange = React.createClass({
    getInitialState: function() {
      return {
        inputValue: ''
      }
    },
    getValue: function() {
      var value = this.refs.nameInput.value;
      this.setState({"name": value});
    },
    render: function() {
      return (
        <div>
          <input type="text" ref="nameInput" onInput={this.getValue} />
          <span>您输入的名字是：{this.state.name}</span>
        </div>
      )
    }
  });
  ReactDOM.render(
    <InputChange />,
    document.getElementById("example")
  );

源码地址
上面代码将input挂到了一个nameInput引用对象，在组件类中通过this.refs.nameInput引用到该input。

组件生命周期

React跟传统的DOM编程不一样，在React中组件的生命状态都在一个order里面，井井有条，什么阶段执行什么方法都由React去完成，所以会比较清晰。
1、componentWillMount
在组件即将render之前立即调用，只会被调用一次
2、componentDidMount
在组件render之后立即调用，只会被调用一次
3、componentWillReceiveProps
在改变组件的props属性时立即调用，在组件初始化的时候不会被调用
4、shouldComponentUpdate
在组件props或者state状态改变之前调用，在组件初始化或者用forceUpdate方法时不会被调用，可以利用这个方法return false即可阻止componentWillUpdate调用
5、componentWillUpdate
在shouldComponentUpdate调用之后立即调用
6、componentDidUpdate
在componentWillUpdate之后刷新DOM完成之后调用
7、componentWillUnmount
在组件被销毁的时候调用
